0.2.0
- Some odd thing, setting parity bit 1 in the SIO results in communication breaking down or jumbling
  at high data rates. Disabling it (even tough at the PC side it is enabled) fixes the communication.
  It seems SCM also has this configuration and i cannot find any example code that has this bit enabled.
  So, i also disable it and hope for the best...
- Hexloader works. relocatable code. Gets moved over the stack upon execution. Command "hexload" or "hl".
  This can overwrite the program without bootloader. 
- new command "sp", shows the current stack pointer.
